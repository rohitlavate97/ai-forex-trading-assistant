# Feature Guide - Technical Analysis Agent

This guide details the mathematical equations, indicators context, tool bindings, system prompts, grounding rules, and verification setups for the Technical Analysis Specialist Agent.

---

## 1. Business Requirement

*   **Intelligent Trend Evaluation**: Automate the calculation and interpretation of price indicators to help traders gauge market trends, identify levels, and evaluate volatility.
*   **Decoupled Calculations**: Isolate financial algorithms from agent prompt text. Math is executed in Python, and raw outputs are supplied as grounded inputs to the LLM.
*   **Probabilistic Safety**: The agent must explain market states in terms of probabilities, avoiding predictions, and must append a standard AI disclaimer on every response.

---

## 2. Financial & Domain Context

The agent is equipped to interpret the following indicators:

### 1. Relative Strength Index (RSI)
*   *Purpose*: Momentum oscillator measuring speed and change of price movements.
*   *Ranges*: Scales between 0 and 100. Traditionally, values $>70$ indicate a currency is overbought (probability of bearish consolidation/reversal), while values $<30$ suggest oversold (probability of bullish consolidation/reversal).

### 2. Moving Average Convergence Divergence (MACD)
*   *Purpose*: Trend-following momentum indicator showing the relationship between two EMAs.
*   *Components*:
    *   **MACD Line**: `EMA_12 - EMA_26`.
    *   **Signal Line**: `EMA_9` of the MACD Line.
    *   **Histogram**: `MACD_Line - Signal_Line`.
    *   *Interpretation*: Crossovers of MACD and Signal lines suggest momentum shifts.

### 3. Bollinger Bands
*   *Purpose*: Volatility bands placed above and below a simple moving average.
*   *Math*: Middle Band = 20 SMA. Upper/Lower bands are shifted by 2 standard deviations.
*   *Interpretation*: Bands contract during low volatility and expand during high volatility. Prices tend to bounce off outer bands.

### 4. Average True Range (ATR)
*   *Purpose*: Measure market volatility.
*   *Math*: Wilder's smoothed average of True Ranges (TR = max of high-low, high-prev_close, low-prev_close). Higher ATR suggests increased volatility.

### 5. Fibonacci Retracement
*   *Purpose*: Horizontal lines indicating where support or resistance is likely to occur.
*   *Math*: Retracement levels computed at key ratios (23.6%, 38.2%, 50.0%, 61.8%, 78.6%) between a major high and low.

### 6. Ichimoku Cloud
*   *Purpose*: Define support/resistance, trend direction, and momentum.
*   *Components*:
    *   **Tenkan-sen (Conversion Line)**: (9-period high + 9-period low) / 2
    *   **Kijun-sen (Base Line)**: (26-period high + 26-period low) / 2
    *   **Senkou Span A (Leading Span A)**: (Tenkan + Kijun) / 2
    *   **Senkou Span B (Leading Span B)**: (52-period high + 52-period low) / 2
    *   **Chikou Span (Lagging Line)**: Close plotted 26 periods behind.

## 4. System Prompt and Guardrails

The agent's system prompt is versioned in [backend/src/modules/agents/tech_analysis/prompts.py](file:///D:/Projects/Full%20Stack%20apps/ai-forex-trading-assistant/backend/src/modules/agents/tech_analysis/prompts.py).

### Enforced Rules:
*   **No Auto-Action**: Explicitly forbids making trade placements.
*   **Strict Disclaimers**: Every response must end with the warning:
    > [!IMPORTANT]
    > **AI Disclosure**: This analysis is generated by an AI assistant for informational and educational purposes only. It is probabilistic in nature and does not constitute financial or investment advice. Trading foreign exchange carries high risk.
*   **Data Grounding**: The agent must reference calculated values. If a tool returns insufficient data, the agent must decline to answer rather than speculating.

## 7. Common Mistakes

*   **Executing Math in Prompt**: Attempting to ask the LLM to count gains/losses and calculate RSI directly.
    *   *Result*: Severe mathematical hallucinations, as LLMs cannot reliably do rolling averages on large number arrays.
    *   *Solution*: Perform calculations in Python (`calculators.py`) and feed the static results to the agent.
*   **Missing Models Imports**: Neglecting to import specific model schemas inside tool functions.
    *   *Solution*: Use clean dictionaries or Pydantic models for tool return signatures.

## 8. Interview Questions

1.  **Why do we decouple indicator calculations from the LLM prompt?**
    *   *Answer*: LLMs are sequence predictors, not calculators. Asking an LLM to calculate a 14-period smoothed Wilder's RSI on a list of float numbers results in massive mathematical errors. Decoupling ensures that Python performs fast, deterministic calculations, and the LLM focuses on what it does best: interpreting the meaning of those results and explaining them to the user.
2.  **How do you enforce probabilistic boundaries in financial AI outputs?**
    *   *Answer*: We enforce strict guidelines in the system instructions, defining acceptable phrasing (e.g., "probability," "potential support") and banning speculative terms (e.g., "will hit," "guaranteed profit"). Additionally, we append a mandatory markdown warning banner to all generated market evaluations.
